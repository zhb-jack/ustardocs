## 二、多端支持的架构证据

### 2.1 核心框架：Expo (React Native) + react-native-web

| 依赖项 | 版本 | 作用 |
|--------|------|------|
| `expo` | ~54.0.33 | 统一的跨平台开发框架，同时支持 iOS / Android / Web |
| `react-native` | 0.81.5 | iOS + Android 原生渲染引擎 |
| `react-native-web` | ^0.21.0 | **关键依赖**，将 React Native 组件映射为 Web DOM 元素 |
| `react-dom` | 19.1.0 | Web 端渲染所需 |

> 这是典型的 **"Write once, run anywhere"** 架构：React Native 负责 iOS/Android 原生渲染，`react-native-web` 负责将同一套组件翻译成浏览器能理解的 HTML/CSS。

### 2.2 app.json 三端配置

```json
"ios": {
  "supportsTablet": true,
  "bundleIdentifier": "com.ustar.app"
},
"android": {
  "adaptiveIcon": { ... },
  "package": "com.ustar.app"
},
"web": {
  "favicon": "./assets/favicon.png",
  "bundler": "metro"
}
```

**三个平台的配置全部齐全**，包括：
- **iOS**：已配置 Bundle ID (`com.ustar.app`)，支持 iPad
- **Android**：已配置包名 (`com.ustar.app`)、自适应图标、边到边显示
- **Web**：已配置 favicon，使用 Metro 作为打包器

### 2.3 package.json 构建脚本

```json
"scripts": {
  "start": "expo start",
  "android": "expo start --android",
  "ios": "expo start --ios",
  "web": "expo start --web",
  "build": "npx expo export --platform web"
}
```

三端的开发和构建命令**全部配置完毕**。

### 2.4 Web 端部署方案 (Vercel)

`vercel.json` 已配置：
- Web 构建命令 (`npx expo export --platform web`)
- SPA 路由重写 (`rewrites`)
- 静态资源长缓存 (`Cache-Control: immutable`)

> 说明 Web H5 端不仅能跑，而且已经准备好了**生产级部署方案**。

## 三、各端运行方式

| 平台 | 启动命令 | 构建/发布方式 |
|------|---------|-------------|
| **iOS** | `npm run ios` | 通过 `eas build --platform ios` 构建 IPA，上传 App Store |
| **Android** | `npm run android` | 通过 `eas build --platform android` 构建 APK/AAB，上传 Google Play |
| **Web H5** | `npm run web` | 通过 `npm run build` 导出静态文件，部署到 Vercel / Nginx 等 |

## 四、跨平台兼容性风险点

虽然架构上三端都支持，但以下几个依赖在 Web 端存在**不同程度的兼容限制**：

| 依赖 | iOS/Android | Web H5 | 风险等级 |
|------|------------|--------|---------|
| `expo-local-authentication` (指纹/面容) | ✅ 完美支持 | ❌ Web 不支持原生生物识别 | 🟡 中 |
| `expo-secure-store` (安全存储) | ✅ Keychain/Keystore | ⚠️ Web 降级为 localStorage | 🟡 中 |
| `@stripe/stripe-react-native` | ✅ 原生 SDK | ⚠️ Web 需改用 Stripe.js | 🟡 中 |
| `react-native-mmkv` (高性能存储) | ✅ 原生模块 | ❌ Web 不支持，需 polyfill | 🔴 高 |
| `expo-sensors` (传感器) | ✅ 完美支持 | ❌ 部分传感器 Web 不可用 | 🟢 低 |
| `react-native-reanimated` (动画) | ✅ 完美支持 | ⚠️ Web 部分动画表现不同 | 🟢 低 |
| `expo-blur` (毛玻璃效果) | ✅ 原生渲染 | ⚠️ Web 降级为 CSS backdrop-filter | 🟢 低 (已用 CSS 兜底) |

> `global.css` 中已经写了 `.glass-panel` 的 `backdrop-filter` CSS 方案，说明开发者**已有意识**地为 Web 端做了视觉降级兜底。

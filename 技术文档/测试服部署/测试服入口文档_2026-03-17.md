## 5. App 与客服链路说明

当前测试服已验证：

1. App 发消息 -> `auth-service`
2. `auth-service` 转发 -> `TGO`
3. TGO 侧创建会话并分配给 `admin`
4. TGO 客服回复 -> 回调到 `auth-service`
5. App 历史消息接口可查到客服回复

当前客服回消息依赖的关键配置：

1. `tgo-platform` 已接入共享网络 `snow-tgo-net`
2. `custom` 平台回调地址为：
   - `http://ustarpay-kong/auth/v1/app/chat/callback`
3. `custom` 平台配置中已同步：
   - `platform_api_key`

说明：

1. 当前客服回消息写入表：`nh_wallet.app_chat_messages`
2. 当前 App 会话表：`nh_wallet.app_chat_sessions`

## 6. Navicat / 数据库连接方式

### 6.1 UStarPay 数据库（标准方式：SSH 隧道）

当前测试服数据库不建议开放公网端口，统一建议通过 SSH 隧道连接。

Navicat 推荐配置：

- 连接类型：`PostgreSQL`
- 主机：`127.0.0.1`
- 端口：`15432`
- 数据库名：`nh_wallet`
- 用户名：`postgres`
- 密码：`postgres`
- SSL：关闭

SSH 页签配置：

- SSH 主机：`164.68.124.12`
- SSH 端口：`22`
- SSH 用户：`root`
- 认证方式：密码

说明：

1. Navicat 通过 SSH 登录服务器后，再访问服务器本机的 `127.0.0.1:15432`。
2. 这样不会把数据库直接暴露到公网。
3. 如果要查 `postgres` 默认库，连接成功后再切换数据库即可。

### 6.2 TGO 数据库

如果需要单独查看 TGO 数据：

- 数据库主机：`127.0.0.1`
- 数据库端口：`5432`
- 数据库名：`tgo`
- 用户名：`tgo`
- 密码：`tgo`

说明：

1. TGO 数据库当前未单独开放公网端口。
2. 如需连接，建议走 SSH 隧道。

# Git 提交规范

## 1. 目的

统一 `D:\snow` 工作区内各仓库的 Git 提交信息口径，避免出现中英文混用、描述不清、协作追踪困难的问题。

## 2. 默认规则

1. 后续默认所有 `git commit message` 必须使用中文。
2. 除非张哥明确要求，或仓库本身存在强制英文提交规范，否则不要使用英文提交信息。
3. 提交信息应直接说明本次改动目标，避免使用空泛描述。

## 3. 推荐写法

1. 优先写成“动词 + 目标”的中文提交，例如：
   - `修复 提现风控最小提现配置`
   - `优化 下载页跳转逻辑`
   - `补充 提现预览限额字段`
2. 如需区分模块，可写成“模块: 动作 + 目标”，例如：
   - `后台提现: 修复最小提现额度展示`
   - `App充值: 修复地址复制按钮`

## 4. 适用范围

1. `ustarpay-app`
2. `ustarpay-backend-go`
3. `ustarpay-frontend-admin`
4. `ustarpay-website`
5. `ustarpay-db`
6. `ustarpay-ops`
7. `D:\snow` 下后续新增并纳入协作的仓库

## 5. 执行要求

1. 该规则从现在起生效，后续新提交统一按中文执行。
2. 历史中已经存在的英文提交不强制回改，避免影响共享分支协作。

# TGO 本地环境变量说明（2026-03-16）

为了避免把本地联调用的 TGO 平台 Key 写进业务配置文件，`auth-service` 现在支持优先读取以下环境变量：

- `NH_TGO_PLATFORM_API_KEY`
- `NH_TGO_BASE_URL`
- `NH_TGO_TIMEOUT`

## 推荐设置方式

```powershell
setx NH_TGO_PLATFORM_API_KEY "你的本地 TGO Platform Key"
setx NH_TGO_BASE_URL "http://127.0.0.1:31234/api/v1"
setx NH_TGO_TIMEOUT "15"
```

## 说明

1. `setx` 会写入当前 Windows 用户环境变量。
2. 新启动的 `auth-service` 会自动读取这些值。
3. `D:\snow\ustarpay-backend-go\configs\auth-service\development.yml` 里不再需要保存真实平台 Key。
4. 如果只是本机临时调试，也可以先在当前终端设置：

```powershell
$env:NH_TGO_PLATFORM_API_KEY="你的本地 TGO Platform Key"
$env:NH_TGO_BASE_URL="http://127.0.0.1:31234/api/v1"
$env:NH_TGO_TIMEOUT="15"
```

## 当前实现

- 配置文件默认值仍保留 `tgo.base_url`
- `tgo.platform_api_key` 已清空
- 代码会优先使用环境变量覆盖
